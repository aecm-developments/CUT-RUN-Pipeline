# CUT&RUN / ChIP-style NGS Processing Pipeline

This repository contains a simple, command-line CUT&RUN-style processing pipeline script (`CUTandRUN_pipeline.sh`) that performs:

1. Adapter trimming + QC (**Trim Galore** + **FastQC**)
2. Alignment (**bowtie2**) to one of:
   - Human genome (hg38)
   - Viral genome (example shown: MCV)
   - *E. coli* spike-in genome
3. SAM → sorted BAM + indexing (**samtools**)
4. Peak calling (**MACS2**)
5. Track generation (**deepTools bamCoverage** → BigWig)
6. Visualization (**IGV**)

> This README documents the workflow and how to adapt it to your dataset.  
> The script is currently written as a *template* (you’ll replace placeholders like `sample` and `directory` with real names/paths).

---

## Inputs

### FASTQ naming
Paired-end reads are expected as two gzipped FASTQ files per sample:

- `SAMPLE_R1.fastq.gz`
- `SAMPLE_R2.fastq.gz`

The example commands in the script use `sample.R1.fastq.gz` / `sample.R2.fastq.gz`. Either style is fine—just be consistent and update the commands accordingly.

---

## Outputs (per sample)

Typical outputs generated by this workflow:

- **Trimmed FASTQs** (Trim Galore output directory)
- Alignment:
  - `sample-*.sam`
  - `sample-sorted.bam`
  - `sample-sorted.bam.bai`
  - `bowtie_log.txt`
  - `sample-metrics.txt` (bowtie2 metrics)
  - `samtools idxstats` summary
- Peak calling (MACS2):
  - `sample_peaks.narrowPeak` (and related MACS2 outputs)
- Tracks:
  - `sample.bw` (BigWig for IGV)

## Reference genome prep (one-time setup)

You must build bowtie2 indexes for each reference you plan to align to.

### Human (hg38)
Download an hg38 FASTA (example file name from NCBI):
`hg38_GrCh38.p14_genomic.fna`

Build the index:

```bash
bowtie2-build hg38_GrCh38.p14_genomic.fna hg38
```

This generates files like `hg38.1.bt2`, `hg38.2.bt2`, etc.

### Viral genome (example: MCV)
Obtain the virus FASTA, then build:

```bash
bowtie2-build MCV.fa MCV
```

> Note: the script comments mention “Use MCV2 for NCCR in the middle of sequence.”  
> If you use a custom virus reference (e.g., rearranged NCCR), build an index for *that exact FASTA*.

### *E. coli* spike-in
Download the *E. coli* FASTA (the script notes the Epicypher link) and build:

```bash
bowtie2-build ecoli.fa ecoli
```

If the download is a `.tar.gz`, unpack first:

```bash
tar -xvzf filename.tar.gz
```

## Step-by-step workflow

### 1) Adapter trimming + QC (Trim Galore + FastQC)

Activates `trim` conda env, trims adapters/low-quality bases, and runs FastQC.

```bash
conda activate trim

trim_galore \
  --paired \
  --fastqc \
  --quality 20 \
  --length 30 \
  --cores 25 \
  -o <OUTPUT_DIR> \
  <SAMPLE_R1.fastq.gz> \
  <SAMPLE_R2.fastq.gz>

conda deactivate
```

**Key parameters**
- `--quality 20`: trims low-quality ends below Q20
- `--length 30`: drops reads shorter than 30 bp after trimming
- `--cores 25`: parallelism (tune to your machine)

---

### 2) Alignment (bowtie2)

The script includes three alignment modes. Choose the one you need (human, virus, or *E. coli*).

#### Human (hg38) — local alignment
```bash
bowtie2 --very-sensitive-local -p 25 -x hg38 \
  -1 <TRIMMED_R1.fastq.gz> \
  -2 <TRIMMED_R2.fastq.gz> \
  -S sample-human.sam \
  -t \
  --met-file sample-metrics.txt \
  2> bowtie_log.txt
```

#### Virus (MCV example) — end-to-end alignment
```bash
bowtie2 --very-sensitive -p 25 -x MCV \
  -1 <TRIMMED_R1.fastq.gz> \
  -2 <TRIMMED_R2.fastq.gz> \
  -S sample-MCV.sam \
  -t \
  --met-file sample-metrics.txt \
  2> bowtie_log.txt
```

#### *E. coli* spike-in — end-to-end alignment
```bash
bowtie2 --very-sensitive -p 25 -x ecoli \
  -1 <TRIMMED_R1.fastq.gz> \
  -2 <TRIMMED_R2.fastq.gz> \
  -S sample-ecoli.sam \
  -t \
  --met-file sample-metrics.txt \
  2> bowtie_log.txt
```

**Notes**
- `--met-file` captures bowtie2 metrics in a separate file; stderr goes to `bowtie_log.txt`.
- Tune `-p` threads to your resources.

---

### 3) SAM → sorted BAM + indexing (samtools)

```bash
samtools view -@ 25 -b sample.sam \
  | samtools sort -@ 25 -o sample-sorted.bam

samtools index sample-sorted.bam
samtools idxstats sample-sorted.bam
```

`idxstats` gives a quick per-reference summary that’s useful for sanity checks (e.g., spike-in abundance).

---

### 4) (Optional) Normalization using *E. coli* spike-in

The script contains a placeholder comment:

> “Normalization of data from Ecoli spike in DNA- MOST RECENT DATA IS EVEN”

Common approaches include:
- Scaling coverage tracks by spike-in read counts
- Using deepTools `--scaleFactor` when generating BigWigs
- Using external scripts to compute a per-sample scaling factor from *E. coli* alignment counts


---

### 5) Peak calling (MACS2)

Activates `ChIPseq` env and calls peaks. Choose the genome size appropriate to your reference.

```bash
conda activate ChIPseq
```

#### Virus
```bash
macs2 callpeak \
  -t sample-sorted.bam \
  -f BAMPE \
  -n sample \
  -g 6000 \
  --nomodel \
  --keep-dup all \
  -q 0.01 \
  --call-summits
```

#### Human
```bash
macs2 callpeak \
  -t sample-sorted.bam \
  -f BAMPE \
  -n sample \
  -g 2.7e9 \
  --nomodel \
  --keep-dup all \
  -q 0.01 \
  --call-summits
```

```bash
conda deactivate
```

**Notes**
- `-f BAMPE` indicates paired-end BAM.
- `--nomodel` is commonly used in CUT&RUN-style data; keep if it matches your lab’s standard.
- `-g` (“effective genome size”) should match your reference. The values above follow the script; adjust as needed.

---

### 6) BigWig generation (bamCoverage)

These tracks can be loaded into IGV.

#### Virus
```bash
bamCoverage \
  -b sample-sorted.bam \
  -o sample.bw \
  --binSize 1 \
  --normalizeUsing None \
  -p 25 \
  --verbose \
  --centerReads
```

#### Human
```bash
bamCoverage \
  -b sample-sorted.bam \
  -o sample.bw \
  --binSize 10 \
  --normalizeUsing CPM \
  -p 25 \
  --verbose \
  --centerReads
```

**Notes**
- `--normalizeUsing CPM` normalizes to “counts per million mapped reads.”
- Bin size is smaller for viral genomes in the template; tune per your needs.

---

### 7) Visualization (IGV)

Launch IGV and load the relevant genome + tracks:

```bash
igv
```

Load:
- the reference genome (hg38 or your virus/*E. coli*)
- `sample.bw` tracks
- optionally, the BAM for read-level inspection

## Troubleshooting tips

- **Trim Galore output names**: Trim Galore renames files; verify the exact trimmed FASTQ filenames before alignment.
- **Alignment looks wrong**: confirm you used the correct `-x` index prefix and that your FASTQ pairs match.
- **Very low mapped reads**: check adapter trimming reports, FastQC summaries, and that you’re aligning to the right reference.
- **MACS2 peaks look odd**: verify genome size (`-g`) and whether you need a control sample for your specific experiment.
